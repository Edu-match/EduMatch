## 1. 改善された項目

### 1.1 AIペルソナの法的安全機構の汎用化と強化

| 比較項目 | 改善前（7/5） | 改善後（7/6） |
|------|------|------|
| 入力制限 | 歴史上の人物名のみ | 人物名 + 自由記述（概念・架空キャラも可） |
| 法的チェック対象 | 実在人物のみ | 実在人物 + 著作権保護キャラクター + 架空の概念 |
| 判定粒度 | `living` + `status` | `living` + `realPerson` + `status`（3軸） |
| 著作権保護キャラ | 判定なし（すり抜け可能） | `realPerson=false, status="blocked"` で明示ブロック |
| 肖像権ポップアップ | チェックボックス＋非表示の details | 明示的なモード切替＋視覚的に目立つ確認フロー |
| 名称形式 | 任意（管理者の手動判断） | **全経路で `AI○○` を自動強制**（なりすまし防止） |
| システムプロンプト | 生成後は内部に隠蔽 | **生成後に表示＋管理者が編集可能**（透明性向上） |
| 編集不可ルール | なし | 固定ルール（建設的発言・説教禁止・AI非開示）が必ず付加 |

**セキュリティ上の意義**:
- 「AI○○」強制により、プラットフォーム上でAI生成の発言と人間の発言が常に区別される（なりすまし・誤認防止）
- 著作権保護キャラクターの自動検出により、権利侵害リスクのあるペルソナが意図せず作成されることを防止
- システムプロンプトの可視化により、「AIが何を指示されているか」が管理者に透明化（ブラックボックス化を回避）
- 編集不可部分の固定により、管理者であっても「AIだと名乗らない」「建設的に」等の安全ルールを削除できない

### 1.2 DBマイグレーション適用とP2022グレースフルデグラデーション

| 比較項目 | 改善前（7/5） | 改善後（7/6） |
|------|------|------|
| Review テーブル | マイグレーション未適用 → 500エラー | 適用完了（post_id, parent_id, author_role カラム追加） |
| Profile テーブル | directory_opt_in 未存在 → 500エラー | 適用完了（directory_opt_in カラム追加） |
| エラーハンドリング | P2022で即500（ページ全体が死亡） | P2022を`.catch()`でキャッチし空データを返す（フォールバック） |
| 影響範囲 | 記事詳細・サービス詳細・マッチングの3ページが完全に利用不可 | 全ページが正常描画（機能のみグレースフル制限） |

**セキュリティ上の意義**:
- 500エラー画面はスタックトレースの露出リスクがある（本番でもNext.jsのエラーバウンダリが内部情報を含み得る）
- グレースフルデグラデーションにより、DBスキーマの不整合がユーザーにエラーとして見えなくなった
- マイグレーション適用により、記事レビュー機能（口コミ + AI返信スレッド）が正常稼働し、データ整合性が確保された

### 1.3 ナビゲーション復活によるUIの信頼性向上

| 比較項目 | 改善前 | 改善後 |
|------|------|------|
| 広場（/forum）のナビゲーション | 「没入モード」でセクションナビ非表示 | 常時表示（サイト共通chrome） |
| ページ間移動 | 広場に入ると他セクションへの導線が消失 | 全11セクションへ常時1クリックで移動可能 |

**セキュリティ上の意義**:
- ナビが消える＝「戻れない」という体験は、フィッシングサイトの典型パターンに近い印象を与え得る
- 一貫したUIクロームの維持はプラットフォームの信頼感（legitimate感）に寄与

## 3. AIペルソナのシステムプロンプト構造

Ryoからの依頼事項：基準となるシステムプロンプトの内容を共有。

### 3.1 固定部分（編集不可・全ペルソナ共通）

```
【以下はシステム固定（編集不可）】
返信ルール: 教育コミュニティにふさわしい建設的な発言をする。断定・説教・宣伝・政治的に偏った発言は避ける。AIだと名乗らない。
```

この固定ルールは `persona-reply.ts` の `generatePersonaReplyText()` 実行時に自動で付加されます。管理者UIでは「これは編集できません」と視覚的に明示されます。

### 3.2 編集可能部分（persona_prompt フィールド）

AIが生成する `persona_prompt` の標準構造（350〜550字）:

```
あなたは「AI○○」として教育のひろばに参加する人物です。

■ 一人称と口調
（例：「僕」を使い、語尾は「〜だよね」「〜かな」など親しみやすい口語。）

■ 大切にする観点・議論スタンス
（例：生徒の自主性を最優先に考える。制度論より現場の声を重視する。）

■ 引き合いに出す経験・活動
（例：「自分が探究学習の授業を作ったときは…」「起業部の活動で感じたのは…」）

■ よく使う言い回し
（例：「それ面白いね」「具体的にはどんな場面で？」と相手に問い返す傾向。）

■ 避ける話題・言い方
（例：特定の政党への言及、断定的な「〜すべき」表現。）
```

### 3.3 実行時に追加される文脈（persona-reply.ts）

```
## あなたの人格
{persona_prompt}

## 大切にしている価値観
{values_text}

## 得意分野
{expertise}

## 参考にできる公的文書・資料の抜粋
{knowledgeContext}（RAGから取得、ある場合のみ）

## 返信ルール
- 上の「あなたの人格」になりきる。一人称・口調・口ぐせを必ず守る
- あなた固有の視点・価値観・経験を最低1つ織り込む
- 相手の投稿にまず具体的に反応してから自分の考えを述べる
- スレッドの流れを踏まえ、既出意見の繰り返しを避けて会話を前に進める
- 100〜220文字程度の自然な口語。AIだと名乗らない
- 断定・説教・宣伝・政治的に偏った発言は避け、建設的に
- 説明や前置きは書かず、返信本文だけを出力する
```
